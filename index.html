计算机网络基础

2019-12-09

[TOC]

一、五层模型

应用层、传输层、网络层、链接层、实体层

1、实体层

物理连接手段，传送01信号

2、链接层

以太网（Ethernet）协议占据主导地位

帧：Frame = Head + Data

Head : 18B

Data : 46~1500B

Head : 包含MAC地址（网卡的地址，48bit，12个16进制）

同一子网内，通过广播和MAC地址传递信息

3. 网络层

IP协议/IP地址

IPv4 : 32bit 网络部分 + 主机部分

子网掩码（subnet mask）：32bit 网络部分为1，主机部分为0

IP数据包 ： 20~60B Head （总共65535B）

3.1 ARP协议

通过IP地址查找MAC地址

  1. 子网内部：地址填：FF:FF:FF:FF:FF:FF（广播地址）
  2. 不在同一子网：把数据包传送给两个子网连接的网关（gateway）

4. 传输层

数据包供哪个程序使用——参数“端口”（port）（0～65535）16个二进制

0～1023系统占用

主机+端口 = 套接字（socket）

4.1 UDP

Head : 8B（总长65535B正好放进一个IP数据包）定义发出端口和接收端口

4.2 TCP

没有长度限制，能确认对方是否收到

5. 应用层

规定应用程序的数据格式

DNS 基础知识

2019-05-26

一、

1、域和区域的区别

域 - 全包括

区域 - 排除已授权出去的子域

名称服务器加载的是区域的数据

2、名称服务器类型

  • Primary master
  • Secondary master(slave)

当slave启动时，它会联系master名称服务器

3.记录信息

  • A记录 记录IPv4地址，TTL默认600s
  • AAAA记录 记录IPv4地址，默认600s
  • CNAME记录 指向域名
  • NS记录 记录授权DNS服务器域名，至少添加2组DNS服务，默认600s

二、DNS server

  1. Recursive resolvers

  2. Root nameservers

    13个 由ICANN监督

  3. TLD nameservers

    1. 通用顶级域名

      • com 商业组织
      • edu 教育组织
      • gov
      • mil 军事组织
      • net 提供网络基础设施/商业组织
      • org 非营利性/商业组织
      • int 国际组织

    2. 国家/地区代码顶级域

      uk,us,ru,jp

  4. Authoritative name servers

三、DNS的三种查询方式

  • 递归查询
  • 迭代查询
  • 非递归查询

1.递归查询（Recursive query）

Client –> DNS Server

2. 迭代查询（Iterative query）

3. 非递归查询（Non-recursive query）

Client –> DNS Server

  1. 浏览器DNS缓存
  2. 操作系统DNS缓存

DNS Server 已经知道答案，直接返回结果

  • DNS Server 本机缓存了对应IP
  • DNS Server 本机缓存了对应的域名的权威服务器

1. DNS使用TCP

  1. 返回响应超过512B（UDP为了避免IP分片的风险，报文尺寸往往小于最小的物理链路X.25的MTU = 576字节，所以512字节是一个安全的尺寸。）
  2. 区域传送

2. 区域传送

区域传送：区域传送是主域名服务器向辅助域名服务器传送变化的那部分数据，辅域名服务器会定时向主域名服务器进行查询，以便了解数据是否有变动。如有变动，会执行一次区域传送，进行数据同步。
